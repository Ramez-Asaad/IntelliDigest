# IntelliDigest UI design system

## 1) Core design intent
This document defines the visual and product-design identity for IntelliDigest: an AI research assistant with a custom web UI—clean, warm, intelligent, and polished rather than generic or cold.

The goal is interfaces that feel custom, clean, warm, intelligent, and well made — never generic, childish, boring, corporate-cold, or obviously AI-generated.

This system is mobile-first, visually polished, and adaptable to desktop.

---

## 2) Brand Personality
### Desired traits
- Clean
- Warm
- Intelligent
- Custom
- Youthful creator energy
- Soft human-centered
- Modern minimal
- Premium professional
- Subtle tech undertone

### Must avoid
- Childish
- Boring
- Cold corporate energy
- Generic AI dashboard feel
- Overly modular sterile interfaces
- Visually empty minimalism
- Reddit-like or default-template layouts

### Emotional statement
The interface should feel like:
**“A phoenix rising from waves instead of ashes.”**
This means renewal, fluid resilience, emotional depth, calm power, and motion with elegance. The system should hint at water, flow, and rebirth rather than flames, aggression, or harshness.

---

## 3) Visual Positioning
### Primary direction
Soft human-centered + modern minimal

### Supporting direction
Premium professional + restrained cyber/tech

### Positioning sentence
A calm, precise, and expressive product language for AI-native tools that feel handcrafted rather than generated.

---

## 4) Color Direction
### Core palette intent
Use cool, fluid, luminous tones. Avoid flat black and muddy warmth. The palette should feel like deep water, pale mist, quiet energy, and clean light.

### Primary palette
- Deep Blue — primary identity anchor
- Aquarius / aqua-cyan — energetic accent
- Light Gray — structural neutral
- Light Green — freshness, success, subtle life

### Recommended implementation roles
- **Primary brand color:** deep blue
- **Primary accent:** aqua / aquarius
- **Secondary accent:** soft light green
- **Base surface:** off-light gray or mist gray
- **Text base:** deep slate-blue rather than pure black

### Avoid
- Pure black-dominant UI
- Dark brown
- Overly saturated rainbow accents
- Harsh red-heavy systems unless used only for destructive actions

### Mood guidance
The palette should feel like:
- ocean light
- glass over water
- intelligent calm
- fresh air
- quiet premium clarity

---

## 5) Theme Strategy
Support both light and dark mode equally.

### Light mode
Should feel airy, layered, premium, and soft — not plain white.
Use tinted light surfaces instead of sterile white.

### Dark mode
Should avoid true black. Use deep blue-charcoal surfaces with luminous aqua and gray highlights.
Dark mode should feel cinematic and fluid, not gamer-neon.

---

## 6) Shape Language
### Corners
- Slightly rounded throughout
- Prefer medium radius over pill shapes
- Avoid overly sharp or overly bubbly components

### Component feel
- Soft precision
- Layered cards
- Gentle depth
- Minimal but not flat

### Interaction philosophy
Touch targets should feel smooth and intentional, with subtle motion and clear tactile hierarchy.

---

## 7) Typography Direction
### Tone
A mix of:
- friendly
- readable
- clean modern

### Typography rules
- Use a modern sans-serif with excellent mobile legibility
- Headlines should feel composed and confident, not loud
- Body text should prioritize rhythm and ease of reading
- Slight size contrast between headers and content for elegance, not drama
- Avoid overly futuristic display fonts
- Avoid sterile enterprise typography

### Typographic personality
Readable enough for productivity tools, but polished enough to feel branded.

---

## 8) Layout Principles
### Mobile-first priorities
1. Visual beauty
2. Gesture feel
3. Fast navigation
4. Quick AI interactions

### Structural rules
- Design for thumb-friendly use first
- Prioritize vertical flow over dense desktop-first layouts
- Use modular stacking that can expand gracefully to desktop
- Keep screen hierarchy obvious within the first glance
- Favor one strong focal action per screen

### Desktop adaptation
- Preserve the mobile emotional feel on desktop
- Expand with whitespace and side panels, not by overcrowding
- Allow multi-column layouts only when they improve clarity

---

## 9) Depth and Texture Rules
Minimal, but not emotionally empty.

### Include
- subtle layering
- tonal surface shifts
- soft shadows or elevated panels
- occasional glass-like or mist-like sections when useful
- restrained gradients inspired by water/light

### Avoid
- flat monochrome emptiness
- harsh neumorphism
- heavy skeuomorphism
- noisy decorative backgrounds

---

## 10) Product Types This System Must Support
- LLM chatbot apps
- Portfolio-like project dashboards
- AI productivity tools

### Frequently recurring screen types
- Dashboards
- Chat interfaces
- Hero sections

---

## 11) App Shell Pattern
### Mobile app shell
- Top area for context or page identity
- Bottom navigation for primary movement
- Main content stack in cards, panels, or message blocks
- Floating contextual actions only when they add speed

### Desktop app shell
- Preserve top identity area
- Optional left rail or slim side navigation
- Main content centered in readable-width panels
- Secondary tools may live in drawers/sheets rather than permanent clutter

---

## 13) Chat Interface Style Rules
Since many future products will involve LLM interaction, chat and prompt surfaces need a distinct identity.

### Desired feel
- Calm intelligence
- Soft precision
- Visually polished, not toy-like
- More “crafted studio product” than “generic chatbot”

### Rules
- Use breathable spacing
- Separate user/system/assistant content clearly without over-styling
- Avoid overly bright bubble colors
- Make the input bar feel premium and tactile
- Use action rows that are subtle but discoverable
- Present long responses in structured blocks for scanning

---

## 14) Dashboard Style Rules
### Desired feel
Not a cold BI tool. Not a corporate admin panel. It should feel editorial, purposeful, and modern.

### Rules
- Use summary cards with strong hierarchy
- Keep widgets limited and meaningful
- Avoid cluttered grids
- Blend data blocks with narrative sections when useful
- Let dashboard screens feel like an overview of meaningful work, not raw metrics overload

---

## 15) Hero Section Rules
Hero sections should feel premium, calm, and a little cinematic.

### Include
- one clear message
- one emotional supporting visual idea
- one primary CTA and one secondary CTA
- layered background or soft gradient atmosphere

### Avoid
- overpacked startup-marketing layouts
- busy illustrations
- generic AI orb visuals

---

## 16) Motion and Interaction
### Motion style
- Fluid
- Soft
- Confident
- Short and intentional

### Inspiration
Think wave motion, drift, reveal, lift, settle.
Not bounce, snap, or flashy neon transitions.

### Interaction rules
- Use motion to clarify hierarchy
- Favor subtle hover/focus/press states
- Make gestures feel elegant on mobile
- Avoid animation for decoration alone

## 18) Do / Don’t Rules
### Do
- Use light, cool, fresh surfaces
- Build obvious hierarchy
- Keep layouts elegant and purposeful
- Let depth come from layers, spacing, and tone
- Make AI interactions feel premium and readable
- Prioritize beauty and flow on mobile

### Don’t
- Don’t make it look enterprise-corporate
- Don’t make it look like an auto-generated SaaS template
- Don’t rely on black-heavy palettes
- Don’t overuse modular card grids with no personality
- Don’t make everything ultra-flat
- Don’t make the interface too playful or juvenile

---

## 19) Reusable Master Prompt for Future Design Generation
Create a mobile-first web app UI that feels custom, warm, intelligent, and exceptionally clean. The visual language should be soft human-centered modern minimal with premium professional polish and a restrained cyber/tech undertone. Use a palette centered on deep blue, aqua-cyan, light gray, and soft light green. Avoid pure black, dark brown, cold corporate styling, childish visuals, generic AI-dashboard patterns, and empty flat minimalism. Use slightly rounded corners, layered depth, subtle gradients, breathable spacing, and fluid motion inspired by water. The emotional metaphor is a phoenix rising from waves instead of ashes — renewal through water, calm strength, and elegant energy. Optimize for mobile beauty, gesture feel, fast navigation, and smooth AI interactions. The result should feel handcrafted, youthful, and high quality.

## 22) Spacing System
Use a consistent 4px base scale.

- xs: 4px
- sm: 8px
- md: 12px
- lg: 16px
- xl: 24px
- 2xl: 32px
- 3xl: 48px

### Rules
- Cards padding: 16–24px
- Section spacing: 32–48px
- Mobile horizontal padding: 16px

---

## 23) Border Radius System
- Small elements: 8px
- Buttons / inputs: 12px
- Cards: 16px
- Large panels / modals: 20px

Avoid pill shapes unless for tags/chips.

---

## 24) Shadow & Depth
Keep shadows soft and layered.

- Subtle: 0 2px 6px rgba(0,0,0,0.04)
- Medium: 0 8px 24px rgba(0,0,0,0.06)
- Elevated: 0 16px 40px rgba(0,0,0,0.08)

Optional: combine with light gradients instead of relying only on shadows.

---

## 25) Typography System
### Suggested Fonts
- Primary: Inter / SF Pro / Geist
- Alternative: Plus Jakarta Sans

### Scale
- H1: 28–32px
- H2: 22–24px
- H3: 18–20px
- Body: 14–16px
- Small: 12–13px

### Rules
- Line height: 1.4–1.6
- Slight letter spacing for headers
- Avoid heavy bold everywhere — use weight contrast strategically

---

## 26) Button System
### Primary Button
- Background: Deep Blue
- Text: White
- Radius: 12px
- Hover: Slight darken + lift

### Secondary Button
- Background: Surface
- Border: Light gray
- Text: Primary color

### Ghost Button
- Transparent
- Text: Secondary
- Hover: subtle background tint

### Interaction
- Add micro-lift (translateY -1px)
- Smooth 150–200ms transitions

---

## 27) Input & Chat Composer
### Input Field
- Background: Surface 2
- Border: subtle
- Radius: 12px
- Padding: 12–14px

### Chat Input Bar
- Slight elevation
- Sticky at bottom (mobile)
- Includes:
  - text input
  - send button
  - optional actions (attach, tools)

### Feel
Should feel like a premium tool, not a messaging app clone.

---

## 28) Card System
### Base Card
- Radius: 16px
- Padding: 16–24px
- Background: Surface
- Border: subtle or none

### Variants
- Elevated card (dashboard highlights)
- Soft card (tinted background)
- Interactive card (hover lift)

### Rule
Every card should have a clear purpose — avoid meaningless grid spam.

---

## 29) Chat Message Blocks
### User
- Right-aligned or neutral block
- Slight tint of primary

### AI
- Left-aligned
- Clean neutral background
- Structured spacing for readability

### Enhancements
- Action row (copy, regenerate)
- Expand/collapse long responses
- Code/structured blocks clearly separated

---

## 30) Navigation System
### Mobile
- Bottom navigation (3–5 items max)
- Floating primary action optional

### Desktop
- Slim sidebar or top nav
- Avoid heavy admin-panel layouts

### Rule
Navigation should feel lightweight and fast.

---

## 31) Motion Tokens
- Duration: 150–250ms
- Easing: ease-out or cubic-bezier(0.22, 1, 0.36, 1)

### Motion Types
- Fade + slight lift
- Slide-in panels
- Soft scale (0.98 → 1)

### Avoid
- Bounce
- Over-elastic motion
- Flashy transitions

---
